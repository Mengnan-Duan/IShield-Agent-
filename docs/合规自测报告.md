## 四、关键 Bug 修复

### Bug 1: numpy 模块找不到（rule_engine 不工作）

- **症状**: `.venv\Scripts\python.exe` 从 `backend/` 运行，`.venv` 路径解析为 `backend\.venv`（不存在），导致 `import numpy as np` 全局失败
- **影响**: `rule_engine.py` 的 `_match_one` 函数内 `import numpy as np` → 所有规则匹配失败
- **修复**: 从 `rule_engine.py` 移除顶层 `import numpy as np`（该模块不需要 numpy）

### Bug 2: NFKC 规范化破坏 SQL 特殊字符

- **症状**: `normalize_input` 使用 `unicodedata.normalize("NFKC", text)` 将 ASCII 单引号 `'` 转为 Unicode 弯引号 `'`
- **影响**: SQL 注入载荷 `' OR '1'='1` 被转为 `' OR '1'='1'`，签名 `' OR '1'='1` 无法匹配
- **修复**: 在 `hybrid_detect` 入口新增弯引号还原逻辑（`normalize_all` 中加入 CURVY_QUOTE_MAP）

### Bug 3: SQL 新规则 KeyError（KeyError: 'keywords'）

- **症状**: `signatures.json` 的 `sql_patterns` 中的 SQL006-010 只有 `pattern` 字段，没有 `keywords`/`sql_ops`，导致 `rule_engine.py` 的 SQL 循环报 KeyError
- **修复**: SQL006-010 移至 `rules[]`（精确子串匹配），`sql_patterns` 保留原有组合规则格式
