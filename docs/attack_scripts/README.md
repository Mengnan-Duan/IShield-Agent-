# IShield 攻击脚本库

> 对应《信息安全大赛》阶段 2.2 · 成果1：安全风险分析报告
> 至少 3 类攻击场景，每类含对抗样本 + 攻击脚本

---

## 目录结构

```
attack_scripts/
├── README.md          # 本文件
├── 01_prompt_injection.py   # 攻击场景1：提示注入
├── 02_tool_hijack.py        # 攻击场景2：工具劫持
├── 03_memory_poison.py      # 攻击场景3：记忆中毒
└── 04_advanced_adversarial.py  # D类高级对抗样本（D-001 ~ D-015）
```

---

## 快速开始

### 前提条件
```bash
cd backend
pip install requests
python app.py          # 启动后端（端口 5000）
```

### 一键运行全部攻击脚本
```bash
cd docs/attack_scripts
python 01_prompt_injection.py   # 提示注入检测
python 02_tool_hijack.py        # 工具劫持检测
python 03_memory_poison.py      # 记忆中毒检测
python 04_advanced_adversarial.py  # 高级对抗样本
```

## 攻击场景详解

### 场景1：提示注入（Prompt Injection）
**文件**: `01_prompt_injection.py`

| 子类 | 描述 | 代表样本 |
|------|------|---------|
| 指令覆盖 | 覆盖系统指令 | A-001 ~ A-005 |
| 角色扮演绕过 | 伪装成无限制AI | A-006 ~ A-009 |
| 系统标签注入 | XML/YAML/JSON标签注入 | A-010 ~ A-012 |
| 编码混淆 | 零宽字符/同形字/Base64 | A-050 ~ A-055 |
| 多语言混合 | 中英日俄多语混合 | A-060 ~ A-062 |

**检测引擎**: 规则引擎（SIG001-SIG015）+ 语义引擎（SEM001-SEM008）

---

### 场景2：工具劫持（Tool Hijacking）
**文件**: `02_tool_hijack.py`

| 子类 | 描述 | 代表样本 |
|------|------|---------|
| SQL注入 | UNION/布尔盲注/堆叠查询 | B-001 ~ B-009 |
| SSRF滥用 | 元数据探测/内网扫描/DNS重绑定 | B-010 ~ B-019 |
| XSS注入 | 反射型/存储型/DOM XSS | B-020 ~ B-024 |
| 路径遍历 | /etc/passwd/SSH密钥/AWS凭证 | B-030 ~ B-039 |
| HTTP注入 | 头部注入/数据外发 | B-040 ~ B-045 |

**检测引擎**: 规则引擎 + 供应链守卫（SupplyChainGuard）

---

### 场景3：记忆中毒（Memory Poisoning）
**文件**: `03_memory_poison.py`

| 子类 | 描述 | 代表样本 |
|------|------|---------|
| 上下文累积 | 渐进式建立权限上下文 | C-001 ~ C-005 |
| 虚假上下文 | 植入虚假记忆/授权 | C-010 ~ C-015 |
| 跨会话攻击 | 利用上下文继承 | C-020 ~ C-025 |
| 提示词泄露 | 自我描述/编码伪装 | C-030 ~ C-039 |

**检测引擎**: 多轮对话分析（Conversation API）+ 意图分析

---

### D类高级对抗样本
**文件**: `04_advanced_adversarial.py`

D 类样本为高度复杂的组合攻击，需多引擎协同检测：

| ID | 名称 | 特点 |
|----|------|------|
| D-001 | 渐进式角色扮演 | 逐步建立信任后引入恶意目标 |
| D-002 | 编码嵌套注入 | Base64→URL编码→零宽字符三重混淆 |
| D-003 | 间接指令覆盖 | 通过讲故事间接触发指令覆盖 |
| D-004 | 情感绑架 | 社会工程利用紧急情况 |
| D-005 | 知识库污染 | 植入虚假知识后引用 |
| D-006 | 多语言混合绕过 | 中/英/日/俄四语种混合 |
| D-007 | 特权升级链 | read→write→http 三步攻击链 |
| D-008 | 上下文窗口溢出 | 长上下文内嵌入隐藏指令 |
| D-009 | 系统提示词逆向 | 间接推断系统提示词 |
| D-010 | 间接指令注入 | 编程方式执行恶意指令 |
| D-011 | 数据投毒攻击 | 诱导引用虚假外部数据 |
| D-012 | 工具调用链混淆 | 分解为多个无害调用 |
| D-013 | 隐私推断攻击 | 统计推断获取敏感信息 |
| D-014 | 模型后门攻击 | 特定触发词激活隐藏行为 |
| D-015 | 提示链级联攻击 | 多轮渐进式渗透 |

## 评分标准

| 指标 | 计算方式 |
|------|---------|
| 检测率 | 检测到的恶意样本 / 总恶意样本 |
| 误报率 | 误判为恶意的正常样本 / 总正常样本 |
| 阻断率 | 被阻断的攻击 / 总攻击 |
| 确认率 | 用户确认放行 / 总待确认 |
